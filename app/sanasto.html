Taustaa

Mobiilikehityksen tärkeimmät termit lyhyesti selitettynä aakkosjärjestyksessä.

Avaintermit

(HTML5) Boilerplate
Verkkosivuston pohja, jolla pystyy nopeasti rakentamaan HTML5-pohjaisen sivuston.
(Twitter) Bootstrap
Alunperin Twitterillä kehitetty komponenttikirjasta verkkosivujen ja sovellusten rakentamiseen, joka sisältää mm. napit, lomakkeet, navigointielementit ja muut käyttöliittymäkomponentit.
Breakpoint
Mukautuvassa verkkosuunnittelussa ne pisteet, joissa sivun muotoilu muuttuu. Breakpointteina voidaan käyttää eri laitteille tyypillisiä näytönleveyksiä (320px kännykälle, 768px tabletille, 1024px desktopille) tai ne voivat perustua sivun designiin ja sisältöön.
CSS3
Uusin version CSS (Cascading style sheet) standardista. CSS:llä määritellään markup-kielillä (kuten HTML) kirjoitettujen dokumenttien ulkoasutyyli (ml. värit, asemointi ja fontit). Sisällön ja ulkoasun erottelu mahdollistaa sisällöm näyttämisen erilailla eri laitteilla.
Device detection
Verkkosivulla vierailevan laitteen ja sen ominaisuuksien (näytön koko, selain ja selainversio, mediatuki, CSS, HTML ja JavaScript-tuki) tunnistaminen esim. HTTP headerissä olevan user agent -tiedon pohjalta. Tunnistamisen perusteella voidaan tarjota laitteelle sopiva versio sivustosta.
Feature detection
Selaimen ominaisuuksien tunnistaminen ja käyttöliittymän räätälöinti selaimessa esim. javascriptillä.
Framework
Sovelluskehitysympäristö / puolivalmisterunko järjestelmän rakentamiseen. Kattaa jossain määrin myös komponenttikirjastot, templatet ja kehitystyökalut mutta ei esimerkiksi tuotteistettuja CMS-järjestelmiä.
Graceful degradation
Mukautuvassa verkkosuunnittelussa käytetty lähestymistapa, jossa aloitetaan suunnittelu ja toteutus päätelaitteesta, jossa on kaikki tarvittavat toiminnallisuudet ja poistetaan toiminnallisuutta siirryttäessä muihin laitteisiin. Vertaa progressive enhancement, jossa edetään päinvastaisessa järjestyksessä.
HTML5
HTML:n nykyisin käytössä olevan version markkinointitermi sekä modernien verkkoteknologioiden yleisnimitys. Tällä viitataan usein mahdollisuuteen rakentaa myös ohjelmiston toimintaa näyttökerrokseen.
HTML5 app
Esimerkiksi sovellus, jonka toiminta on pääosin selaimessa erotuksena perinteisestä verkkosivusta, jossa html ja näkymät koostetaan palvelimella.
Hybrid-sovellus
Mobiilisovellus, jossa näkymät rakennetaan HTML5:llä ja joka paketoidaan natiivisovellukseksi laitteeseen asennettavaksi. Paketointi tehdään yleensä selainkomponenttia käyttäen tai puolivalmisteella kuten PhoneGap tai AppGyver.
Mediakysely
Mediakysely, eli CSS Media Query, jolla CSS-tyylejä kohdistetaan päätelaitteen ominaisuuksien perusteella.
Mobile first
Suunnittelumenetelmä, jossa suunnitellaan mobiilikäyttö ennen desktop-versiota.
Mukautuva verkkosuunnittelu
Verkkopalveluiden suunnittelumenetelmä, jossa verkkopalvelut rakentamaan niin, että päätelaitteista riippumatta käyttäjä saa hyvän käyttökokemuksen. Tämä saavutetaan media queryjen avulla (joilla nuuskitaan päätelaite ja näytön koko) ja skaalaamalla kuvia ja järjestelemällä muita elementtejä sivuilla (kuten navigaatiot, eri palikoiden sijainti sivuilla jne) sekä käyttämällä suhteellisia kokoja absoluuttisten sijaan (esmes leveysmäärityksissä prosentteja pikselien sijaan).
Natiivisovellus
Laitteessa pyörivä sovellus, kuten sähköposti-client tai vaikkapa angry birds-peli. Eli appsi, joka ladataan jostain, kuten app storesta ja hyödyntää laitteen omaa rautaa ja laitteistoa, kuten kameraa ja mahdollistaa siis myös offline-käytön ja paikallisiin resursseihin (osoitekirjat, muisti) kajoamisen.
Progressive enhancement
Strategia, jossa sovelluksen toimitoja lisätään päätelaitteen ominaisuustuen näin mahdollistaessa. Mahdollistaa sen, että palvelu on käytettävissä ihan kaikilla laitteilla, mutta edistyneemät laitteet saavat ihanamman käyttökokemuksen ja rikkaamman ominaisuussetin.
Web app
Sovellus, jota käytetään verkosta html5-toteutuksena (käytännössä usein html5 + javascript).
