Taustaa

Tämän ohjeistuksen tarkoitus on kuvata periaatteet mobiilipalveluiden toteutukselle Helsingin Yliopistolla. Ohjeistus on tarkoitettu järjestelmien omistajille heidän suunnitellessa uusia kehityshankkeita. Tämä ohjeistus ei pyri määrittelemään mitä mobiilissa pitäisi tehdä vaan tuomaan esille näkökulmia, joita olisi hyvä pitää mielessä kun mietitään palveluiden toteutusta eri kanavissa.

Avaintermit

Native app

Laitteessa pyörivä sovellus, kuten sähköposti-client tai vaikkapa angry birds-peli. Eli appsi, joka ladataan jostain, kuten app storesta ja hyödyntää laitteen omaa rautaa ja laitteistoa, kuten kameraa ja mahdollistaa siis myös offline-käytön ja paikallisiin resursseihin (osoitekirjat, muisti) kajoamisen. http://mobilewebbestpractices.com/strategy/dont-confuse-native-and-web/

Web app

Sovellus, jota käytetään verkosta esmes html5-toteutuksena (käytännössä usein html5 + javascript): "Regardless of your choice of native or non-native Web technologies, what makes a Web application a Web application is that it is deployed to, and accessed from, the World Wide Web platform. This means, at its simplest, that your website or application has a URL and is served via HTTP." http://mobilewebbestpractices.com/strategy/dont-confuse-native-and-web/

html5

Html:n uusin versio, joka tukee uusina tageina mm. audio, canvas, video, ja nav. Uusina ominaisuuksina myös local storage sekä responsiivisuutta parantavia ominaisuuksia. Yleisimmin kuitenkin html5-termiä käytetään myös tarkoittamaan kombinaatiota html5+css3+javascript, jolla saadaan monipuolisia web-palveluita toteutettua. http://fi.wikipedia.org/wiki/HTML5 http://www.w3schools.com/html/html5_intro.asp

Responsive web design
Näkökohta verkkopalveluiden suunnitteluun, jossa verkkopalvelut pyritään rakentamaan niin, että päätelaitteista riippumatta käyttäjä saa hyvän käyttökokemuksen. Tämä saavutetaan media queryjen avulla (joilla nuuskitaan päätelaite ja näytön koko) ja skaalaamalla kuvia ja järjestelemällä muita elementtejä sivuilla (kuten navigaatiot, eri palikoiden sijainti sivuilla jne) sekä käyttämällä suhteellisia kokoja absoluuttisten sijaan (esmes leveysmäärityksissä prosentteja pikselien sijaan). "Responsive Web design is the approach that suggests that design and development should respond to the user’s behavior and environment based on screen size, platform and orientation. The practice consists of a mix of flexible grids and layouts, images and an intelligent use of CSS media queries. As the user switches from their laptop to iPad, the website should automatically switch to accommodate for resolution, image size and scripting abilities. In other words, the website should have the technology to automaticallyrespond to the user’s preferences. This would eliminate the need for a different design and development phase for each new gadget on the market." lähde http://en.wikipedia.org/wiki/Responsive_web_design

CSS3

Uusin version CSS (Cascading style sheet) standardista. CSS:llä määritellään markup-kielillä kirjoitettujen dokumenttien ulkoasutyyli. "CSS is designed primarily to enable the separation of document content (written in HTML or a similar markup language) from document presentation, including elements such as the layout, colors, and fonts.[1] This separation can improve content accessibility, provide more flexibility and control in the specification of presentation characteristics, enable multiple pages to share formatting, and reduce complexity and repetition in the structural content (such as by allowing for tableless web design). CSS can also allow the same markup page to be presented in different styles for different rendering methods, such as on-screen, in print, by voice (when read out by a speech-based browser or screen reader) and on Braille-based, tactile devices. It can also be used to allow the web page to display differently depending on the screen size or device on which it is being viewed. While the author of a document typically links that document to a CSS file, readers can use a different style sheet, perhaps one on their own computer, to override the one the author has specified." http://en.wikipedia.org/wiki/Cascading_Style_Sheets

Backbone on MVP-arkkitehtuuriin pohjautuva Javascript-kirjasto yhden-sivun verkkosovellusten rakentamiseen. http://backbonejs.org

MVC

Model-view-controller on arkkitehtuurimalli, jossa erotetaan tiedon esittäminen käyttäjän interaktiosta. http://en.wikipedia.org/wiki/Model–view–controller

PhoneGap

PhoneGap on sovelluskehitysalusta, jolla voidaan tehdä webbiteknologioita hyödyntymällä sovelluksia kaikille alustoille. http://phonegap.com/about/
