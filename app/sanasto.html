Taustaa

Mobiilikehityksen tärkeimmät termit lyhyesti selitettynä aakkosjärjestyksessä.

Avaintermit

(HTML5) Boilerplate

Verkkosivuston pohja, jolla pystyy nopeasti rakentamaan HTML5-pohjaisen sivuston. http://html5boilerplate.com

(Twitter) Bootstrap

Alunperin Twitterillä kehitetty komponenttikirjasta verkkosivujen ja sovellusten rakentamiseen, joka sisältää mm. napit, lomakkeet, navigointielementit ja muut käyttöliittymäkomponentit.

Breakpoint

Mukautuvassa verkkosuunnittelussa ne pisteet, joissa sivun muotoilu muuttuu. Breakpointteina voidaan käyttää eri laitteille tyypillisiä näytönleveyksiä (320px kännykälle, 768px tabletille, 1024px desktopille) tai ne voivat perustua sivun designiin ja sisältöön.

CSS3

Uusin version CSS (Cascading style sheet) standardista. CSS:llä määritellään markup-kielillä (kuten HTML) kirjoitettujen dokumenttien ulkoasutyyli (ml. värit, asemointi ja fontit). Sisällön ja ulkoasun erottelu mahdollista, että erilaisille sisältö näytetään erilailla eri laitteilla.

Device detection

Verkkosivulla vierailevan laitteen ja sen ominaisuuksien (näytön koko, selain ja selainversio, mediatuki, CSS, HTML ja JavaScript-tuki) tunnistaminen esim. HTTP headerissä olevan user agent tiedon pohjalta. Tunnistamisen perusteella voidaan tarjota laitteelle sopiva versio sivustosta. http://docs.webplatform.org/wiki/concepts/Detecting_device_and_browser

Feature detection

Katso device detection. Esimerkiksi HTML5 boilerplaten mukana tuleva Modernizr feature detection kirjasto tarkistaa käyttäjän selaimem HTML5 ja CSS3 tuen.

Framework
Graceful degradation

Mukautuvassa verkkosuunnittelussa käytetty lähestymistapa, jossa aloitetaan suunnittelu ja toteutus päätelaitteesta, jossa on isoin näyttö, eniten toiminnallisuutta, jne. ja poistetaan toiminnallisuutta siirryttäessä muihin laitteisiin. Vertaa progressive enhancement, jossa edetään päinvastaisessa järjestyksessä.

HTML5

HTML:n uusin versio, joka tukee uusina tageina mm. audio, canvas, video, ja nav. Uusina ominaisuuksina myös local storage sekä responsiivisuutta parantavia ominaisuuksia. Yleisimmin kuitenkin html5-termiä käytetään myös tarkoittamaan kombinaatiota html5+css3+javascript, jolla saadaan monipuolisia web-palveluita toteutettua. http://fi.wikipedia.org/wiki/HTML5 http://www.w3schools.com/html/html5_intro.asp Uutta toiminnallisuutta ilman monimutkaista JavaScriptia kuten form validation ja video. Is better suited to writing dynamic applications than previous HTML versions (HTML was originally designed for creating static documents). Has a clearly defined parsing algorithm so that all browsers implementing HTML5 will create the same DOM from the same markup, regardless of validity. This is a massive win for interoperability.

Mukautuva verkkosuunnittelu

Verkkopalveluiden suunnittelumenetelmä, jossa verkkopalvelut rakentamaan niin, että päätelaitteista riippumatta käyttäjä saa hyvän käyttökokemuksen. Tämä saavutetaan media queryjen avulla (joilla nuuskitaan päätelaite ja näytön koko) ja skaalaamalla kuvia ja järjestelemällä muita elementtejä sivuilla (kuten navigaatiot, eri palikoiden sijainti sivuilla jne) sekä käyttämällä suhteellisia kokoja absoluuttisten sijaan (esmes leveysmäärityksissä prosentteja pikselien sijaan). "Responsive Web design is the approach that suggests that design and development should respond to the user’s behavior and environment based on screen size, platform and orientation. The practice consists of a mix of flexible grids and layouts, images and an intelligent use of CSS media queries. As the user switches from their laptop to iPad, the website should automatically switch to accommodate for resolution, image size and scripting abilities. In other words, the website should have the technology to automaticallyrespond to the user’s preferences. This would eliminate the need for a different design and development phase for each new gadget on the market." lähde http://en.wikipedia.org/wiki/Responsive_web_design

Natiivisovellus

Laitteessa pyörivä sovellus, kuten sähköposti-client tai vaikkapa angry birds-peli. Eli appsi, joka ladataan jostain, kuten app storesta ja hyödyntää laitteen omaa rautaa ja laitteistoa, kuten kameraa ja mahdollistaa siis myös offline-käytön ja paikallisiin resursseihin (osoitekirjat, muisti) kajoamisen. http://mobilewebbestpractices.com/strategy/dont-confuse-native-and-web/

Progressive enhancement
Web app

Sovellus, jota käytetään verkosta html5-toteutuksena (käytännössä usein html5 + javascript): "Regardless of your choice of native or non-native Web technologies, what makes a Web application a Web application is that it is deployed to, and accessed from, the World Wide Web platform. This means, at its simplest, that your website or application has a URL and is served via HTTP." http://mobilewebbestpractices.com/strategy/dont-confuse-native-and-web/

Angular.js

Angular.js on Javascript-sovelluskehitysalusta, joka tuo selainpohjaisiin sovelluksiin model-view-controller -toiminnallisuutta. http://en.wikipedia.org/wiki/Model–view–controller

Backbone

Backbone on MVP-arkkitehtuuriin pohjautuva Javascript-kirjasto yhden-sivun verkkosovellusten rakentamiseen. http://backbonejs.org

MVC

Model-view-controller on arkkitehtuurimalli, jossa erotetaan tiedon esittäminen käyttäjän interaktiosta. http://en.wikipedia.org/wiki/Model–view–controller

PhoneGap

PhoneGap on sovelluskehitysalusta, jolla voidaan tehdä webbiteknologioita hyödyntymällä sovelluksia kaikille alustoille. http://phonegap.com/about/
